1 パッケージ構成

ディレクトリ構造 必須
src/main/java以下のパッケージ構造

com.example.taskapp
├── config
│　└── SecurityConfig.java　　　　// Spring Securityの設定（認証・認可、ログイン設定など）
│　└── WebConfig.java　　　　　　// MVC設定（ViewResolverやInterceptorなど）
│
├── controller
│　└── TaskController.java　　　// タスク関連の画面制御（一覧・登録・編集・削除）
│　└── UserController.java　　　// ユーザー管理機能の制御
│　└── AuthController.java　　　// ログイン・ログアウト処理の制御
│
├── service
│　└── TaskService.java　　　　// タスクの業務ロジック（取得・登録・更新・削除）
│　└── UserService.java　　　　// ユーザーの業務ロジック
│　└── AuthService.java　　　　// 認証関連の処理（必要に応じて）
│
├── repository
│　└── TaskMapper.java　　　　// タスクテーブルへのDBアクセス（MyBatis）
│　└── UserMapper.java　　　　// ユーザーテーブルへのDBアクセス（MyBatis）
│
├── entity
│　└── Task.java　　　　　　　// タスクテーブルに対応するエンティティクラス
│　└── User.java　　　　　　　// ユーザーテーブルに対応するエンティティクラス
│
├── form
│　└── TaskForm.java　　　　　// タスク入力フォーム（バリデーション付き）
│　└── UserForm.java　　　　　// ユーザー入力フォーム
│　└── LoginForm.java　　　　// ログイン画面用フォームクラス
│
└── resources
　　└── templates
　　　　├── task
　　　　│　├── list.html　　　// タスク一覧画面
　　　　│　├── form.html　　　// タスク登録・編集画面
　　　　│　└── detail.html　　// タスク詳細画面
　　　　├── user
　　　　│　├── list.html　　　// ユーザー一覧画面
　　　　│　└── form.html　　　// ユーザー登録・編集画面
　　　　└── login.html　　　　// ログイン画面

パッケージの役割 必須
各パッケージの責務を説明
パッケージ 役割 使用するアノテーション
config アプリケーション全体の設定（セキュリティ、MVC設定など） @Configuration, @EnableWebSecurity, @Bean
controller HTTPリクエストの受付、画面遷移制御、レスポンス返却 @Controller, @RequestMapping, @GetMapping, @PostMapping
service 業務ロジックの実装、トランザクション管理、処理の中核 @Service, @Transactional
repository データベースアクセス（CRUD処理、SQL実行） @Mapper, @Repository
entity データベーステーブルと対応するドメインオブジェクト @Data, @Entity（※JPAの場合）
form 画面入力データの受け取り、バリデーション定義 @Data, @NotNull, @NotBlank, @Size, @Email

3 Repository（リポジトリ）

TaskMapper.java 必須
タスクリポジトリのインターフェース定義

┌────────────────────────────────────────────────┐
│ 《Interface》　　　　　　　　　　　　　　　 │
│ TaskMapper　　　　　　　　　　　　　　　 │
│ @Mapper　　　　　　　　　　　　　　　　 │
├────────────────────────────────────────────────┤
│ + selectAll(): List<Task>　　　　　　　 │
│ + selectById(id: Long): Task　　　　　　 │
│ + selectByUserId(userId: Long): List<Task> │
│ + insert(task: Task): void　　　　　　　 │
│ + update(task: Task): void　　　　　　　 │
│ + delete(id: Long): void　　　　　　　　│
│ + search(keyword: String): List<Task>　 │
└────────────────────────────────────────────────┘

メソッド 引数 戻り値 SQL種別 説明
selectAll なし List<Task> SELECT 全タスクを取得する
selectById id: Long Task SELECT IDを指定してタスクを取得する
selectByUserId userId: Long List<Task> SELECT ユーザーIDに紐づくタスク一覧を取得する
insert task: Task void INSERT 新しいタスクを登録する
update task: Task void UPDATE 既存のタスク情報を更新する
delete id: Long void DELETE IDを指定してタスクを削除する
search keyword: String List<Task> SELECT キーワードに一致するタスクを検索する

DI（依存性注入）の仕組み 任意
Spring BootにおけるDIの説明
@Service public class TaskService { private final TaskMapper taskMapper; // コンストラクタインジェクション（推奨） @Autowired public TaskService(TaskMapper taskMapper) { this.taskMapper = taskMapper; } // ビジネスロジックメソッド public List<Task> findAll() { return taskMapper.selectAll(); } }

Spring Bootでは、@Autowiredアノテーションを使用してDIコンテナからBeanを注入します。 コンストラクタインジェクションが推奨されており、テスタビリティが向上します。
