1 システム概要

システムの目的 必須
このシステムが解決する課題と提供する価値

ワークショップ運営スタッフが使用し、予約情報や顧客情報、空き枠を一元管理できるシステムである。予約状況の可視化や変更対応を効率化し、業務負担を軽減するとともに、円滑な運営と顧客満足度の向上を実現する。

フロントエンド ↔ バックエンド ↔ データベースの関係を図示

4 機能詳細仕様

機能ごとに詳細仕様を記載してください 以下のテンプレートを機能の数だけコピーして使用してください。
最低でもCRUD（登録・一覧・更新・削除）とログイン機能の仕様は記載してください。

機能仕様テンプレート（コピーして使用）

機能名：ログイン機能 必須
機能ID：F-001

1. 画面レイアウト

画面中央：ログインフォーム
・ユーザーID入力欄
・パスワード入力欄
・「ログイン」ボタン

画面下部：エラーメッセージ表示エリア（フォーム直下）

2. 入力項目一覧
項目名　データ型　必須　バリデーションルール
ユーザーID　String　必須　未入力不可、最大50文字
パスワード　String　必須　未入力不可、最大100文字

3. 処理フロー
ユーザー操作 → ログインボタン押下 →
システム処理：入力チェック → 認証処理（ID/Password照合） → セッション作成 →
結果表示：ダッシュボードへ遷移 / エラー表示

4. 正常時の動作
・認証成功時、ログイン状態を保持する（セッション管理）
・ダッシュボード画面へ遷移する

5. エラー時の動作
・バリデーションエラー：該当項目下にエラー文言表示（例：必須です）
・認証エラー：フォーム上部に「ユーザーIDまたはパスワードが正しくありません」
・システムエラー：画面上部に「システムエラーが発生しました」

6. 画面遷移先
・成功：ダッシュボード画面
・失敗：ログイン画面（再表示）

機能名：予約一覧機能 必須
機能ID：F-004

1. 画面レイアウト

画面上部：タイトル「予約一覧」、新規登録ボタン
画面上部（検索エリア）：
・予約日（開始日〜終了日）
・予約者名入力欄
・ステータス選択
・「検索」ボタン
・「リセット」ボタン

画面中央：予約一覧テーブル
・予約ID
・予約日
・時間
・予約者名
・人数
・プラン
・ステータス
・操作（詳細 / 編集 / 削除）

画面下部：ページネーション、件数表示

2. 入力項目一覧
項目名　データ型　必須　バリデーションルール
予約日（開始）　Date　任意　日付形式であること
予約日（終了）　Date　任意　開始日≦終了日であること
予約者名　String　任意　最大50文字
ステータス　String　任意　予約済み / 来店済み / キャンセル のいずれか

3. 処理フロー
ユーザー操作 → 検索条件入力 → 「検索」ボタン押下 →
システム処理：入力チェック → 条件に応じて予約データ取得 → 一覧生成 →
結果表示：予約一覧画面に検索結果を表示

4. 正常時の動作
・予約データを一覧表示する
・検索条件に一致するデータのみ表示する
・各行から詳細 / 編集 / 削除へ遷移できる

5. エラー時の動作
・バリデーションエラー：画面上部に「検索条件が不正です」と表示
・検索結果なし：「該当する予約がありません」と表示
・システムエラー：画面上部に「システムエラーが発生しました」と表示

6. 画面遷移先
・「新規登録」：予約登録画面
・「詳細」：予約詳細画面
・「編集」：予約更新画面
・「削除」：予約削除確認画面

機能名：予約登録機能 必須
機能ID：F-005

1. 画面レイアウト

画面上部：タイトル「予約登録」
画面中央：入力フォーム
・予約日

・予約者名
・メールアドレス
・電話番号
・人数

・備考
・「登録」ボタン
・「戻る」ボタン

画面下部：エラーメッセージ表示エリア

2. 入力項目一覧
項目名　データ型　必須　バリデーションルール
予約日　Date　必須　未入力不可、過去日不可
予約時間　Time　必須　未入力不可、営業時間内であること
予約者名　String　必須　未入力不可、最大50文字
メールアドレス　String　必須　メール形式、最大100文字
電話番号　String　必須　未入力不可、半角数字およびハイフン可
人数　Integer　必須　1以上、定員以下
プラン　String　必須　未選択不可、登録済みプランから選択
備考　String　任意　最大500文字

3. 処理フロー
ユーザー操作 → 必要項目入力 → 「登録」ボタン押下 →
システム処理：入力チェック → 空き枠確認 → 予約データ保存 →
結果表示：予約一覧画面へ遷移 / エラー表示

4. 正常時の動作
・入力内容が正しい場合、予約情報をDBへ登録する
・登録完了後、一覧画面へ戻り、完了メッセージを表示する

5. エラー時の動作
・バリデーションエラー：各項目下にエラーメッセージを表示
・空き枠エラー：「選択した時間帯は満席です」と表示
・システムエラー：画面上部に「システムエラーが発生しました」と表示

6. 画面遷移先
・成功：予約一覧画面
・失敗：予約登録画面（再表示）
・「戻る」：予約一覧画面

5 非機能要件

セキュリティ要件 必須
認証・認可・データ保護に関する要件

・認証方式：Spring Securityによるフォーム認証を採用する。

・パスワードの暗号化方式：BCryptPasswordEncoderを用いてハッシュ化する。

・CSRF対策：Spring SecurityのCSRFトークンにより不正リクエストを防止する。

・XSS対策：ThymeleafのHTMLエスケープ機能（th:text）を利用し、入力値をそのまま出力しない。

・SQLインジェクション対策：JPA/Repositoryを利用し、バインド変数で安全にSQLを実行する。

・セッション管理：ログイン時にセッションを発行し、ログアウト時に無効化する。セッションID再生成により固定化攻撃を防ぐ。

対応環境 必須
対応ブラウザ・デバイス

・対応ブラウザ：Chrome（最新版）/ Firefox（最新版）/ Edge（最新版）
・対応デバイス：PC / タブレット / スマートフォン
