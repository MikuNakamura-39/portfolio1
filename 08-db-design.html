2 テーブル定義

テーブルごとに定義表を作成してください 以下のテンプレートをテーブルの数だけコピーして使用してください。
テーブル名：reservations 必須
テーブルの目的：ワークショップの予約情報を管理する
カラム名 データ型 NULL デフォルト 制約 説明
id BIGINT NO AUTO_INCREMENT PK 予約ID
customer_id BIGINT NO — FK (customers.id) 顧客ID
schedule_id BIGINT NO — FK (schedules.id) スケジュールID
plan_id BIGINT NO — FK (plans.id) プランID
staff_id BIGINT YES NULL FK (staffs.id) 担当スタッフID
reservation_status VARCHAR(30) NO reserved NOT NULL 予約状態
participant_count INT NO 1 NOT NULL 参加人数
total_price DECIMAL(10,2) NO 0.00 NOT NULL 合計金額
note TEXT YES NULL — 備考
reserved_at DATETIME NO CURRENT_TIMESTAMP — 予約日時
created_at DATETIME NO CURRENT_TIMESTAMP — 作成日時
updated_at DATETIME NO CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP 更新日時
deleted_at DATETIME YES NULL — 論理削除日時
テーブル名：customers 必須
テーブルの目的：顧客情報を管理する
カラム名 データ型 NULL デフォルト 制約 説明
id BIGINT NO AUTO_INCREMENT PK 顧客ID
name VARCHAR(50) NO — NOT NULL 顧客名
email VARCHAR(100) NO — UNIQUE メールアドレス
phone VARCHAR(20) NO — — 電話番号
created_at DATETIME NO CURRENT_TIMESTAMP — 作成日時
updated_at DATETIME NO CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP 更新日時
deleted_at DATETIME YES NULL — 論理削除日時
テーブル名：schedules 必須
テーブルの目的：ワークショップの開催枠を管理する
カラム名 データ型 NULL デフォルト 制約 説明
id BIGINT NO AUTO_INCREMENT PK スケジュールID
workshop_date DATE NO — NOT NULL 開催日
start_time TIME NO — NOT NULL 開始時間
end_time TIME NO — NOT NULL 終了時間
capacity INT NO 1 — 定員
created_at DATETIME NO CURRENT_TIMESTAMP — 作成日時
updated_at DATETIME NO CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP 更新日時
deleted_at DATETIME YES NULL — 論理削除日時
テーブル名：staffs 必須
テーブルの目的：スタッフ情報および認証情報を管理する
カラム名 データ型 NULL デフォルト 制約 説明
id BIGINT NO AUTO_INCREMENT PK スタッフID
name VARCHAR(50) NO — NOT NULL スタッフ名
email VARCHAR(100) NO — UNIQUE メールアドレス
password VARCHAR(255) NO — NOT NULL パスワード（ハッシュ）
role VARCHAR(20) NO ROLE_STAFF — 権限
created_at DATETIME NO CURRENT_TIMESTAMP — 作成日時
updated_at DATETIME NO CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP 更新日時
deleted_at DATETIME YES NULL — 論理削除日時

3 インデックス設計

インデックス一覧 必須
テーブル名 インデックス名 対象カラム 種類 設定理由
customers uq_customers_email email UNIQUE 顧客メールアドレスの重複登録を防止するため
customers idx_customers_name name INDEX 顧客名での検索に頻繁に使用するため
customers idx_customers_phone phone INDEX 電話番号での顧客検索に使用するため
staffs uq_staffs_email email UNIQUE スタッフのログイン・識別用メールアドレスの重複を防ぐため
staffs idx_staffs_name name INDEX スタッフ名検索に使用するため
staffs idx_staffs_role role INDEX 権限別の絞り込みに使用するため
admins uq_admins_login_id login_id UNIQUE 管理者ログインIDの重複を防止するため
plans idx_plans_plan_name plan_name INDEX プラン名検索や一覧表示で利用するため
plans idx_plans_is_active is_active INDEX 有効なプランのみを絞り込む際に利用するため
plan_contents idx_plan_contents_plan_id plan_id INDEX 外部キー結合およびプランごとの内容取得に利用するため
plan_contents idx_plan_contents_sort_order sort_order INDEX 表示順で並び替えるため
schedules idx_schedules_workshop_date workshop_date INDEX 開催日での検索に頻繁に使用するため
schedules idx_schedules_status schedule_status INDEX 受付状態での絞り込みに利用するため
schedules idx_schedules_date_time workshop_date, start_time INDEX 日付順・開始時間順の並び替えおよび検索に利用するため
staff_schedules idx_staff_schedules_staff_id staff_id INDEX 外部キー結合およびスタッフごとの担当枠取得に利用するため
staff_schedules idx_staff_schedules_schedule_id schedule_id INDEX 外部キー結合およびスケジュールごとの担当者取得に利用するため
staff_schedules uq_staff_schedules_staff_schedule staff_id, schedule_id UNIQUE 同じスタッフを同じスケジュールに重複登録しないため
reservations idx_reservations_customer_id customer_id INDEX 外部キー結合および顧客ごとの予約検索に利用するため
reservations idx_reservations_schedule_id schedule_id INDEX 外部キー結合およびスケジュールごとの予約検索に利用するため
reservations idx_reservations_plan_id plan_id INDEX 外部キー結合およびプラン別の集計に利用するため
reservations idx_reservations_staff_id staff_id INDEX 担当スタッフ別の予約検索に利用するため
reservations idx_reservations_status reservation_status INDEX 予約状態による絞り込みに利用するため
reservations idx_reservations_reserved_at reserved_at INDEX 予約日時順の並び替えや期間検索に利用するため
reservations idx_reservations_schedule_status schedule_id, reservation_status INDEX 特定スケジュール内で予約状態ごとに検索・集計するため
dashboard_notices idx_dashboard_notices_created_by created_by INDEX 外部キー結合および作成者別の管理に利用するため
dashboard_notices idx_dashboard_notices_published published_flg INDEX 公開中のお知らせのみを抽出するため
audit_logs idx_audit_logs_admin_id admin_id INDEX 外部キー結合および管理者ごとの操作履歴取得に利用するため
audit_logs idx_audit_logs_created_at created_at INDEX 操作日時順での検索・ソートに利用するため
audit_logs idx_audit_logs_target target_table, target_id INDEX 対象データ単位での操作履歴検索に利用するため
