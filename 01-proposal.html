1 プロジェクト概要

サービス名 必須
制作するWebサービスの名前

【Scentory（セントリー）】

由来：Scent（香り）＋ Story（物語）

コンセプト 必須
サービスを一言で表す説明（キャッチコピー＋1〜2文の説明）

【香りでつくる私】

自分だけの香りをブレンドし、物語のように思い出を残せる香水作り体験。初心者でも気軽に参加でき、デートや女子旅にもぴったりの新しい香りの時間を提供します。

ターゲットユーザー 必須
このサービスを使う主なユーザー像（概要レベル）

【大人のカップル・体験重視タイプ】

・年齢層

27〜35歳

・職業

会社員（営業・企画・IT・公務員など）／看護師／フリーランス

・このサービスを使う理由

記念日や旅行の思い出を特別な形で残したい。既製品ではなく、2人だけのオリジナルの香りを作りたい。上質で落ち着いた体験を楽しみたい。

・主な利用シーン

記念日デート／誕生日デート／旅行中の体験イベント／自分たちへのご褒美時間

2 背景と課題

市場の現状 必須
このサービスが存在する市場の現状と課題

香水市場はコロナ後の外出機会回復により再成長傾向にあり、Z世代を中心に「ニッチフレグランス」やジェンダーレス香水、体験型サービスが注目されている。一方で、ユーザーは「自分に合う香りが分からない」「価格が高い」「香りが強すぎる」といった不安を抱えることが多い。既存サービスはブランド主導の販売が中心で、初心者が気軽に試せる機会や、個別提案型の体験が不足している点が課題とされる。

解決すべき課題 必須
本プロジェクトで解決を目指す具体的な課題（3つ程度）

■ 課題①：自分に合う香りが分からない

・現状の問題点（なぜ課題なのか）

既製品は種類が多く、試しても本当に自分に合うか判断しづらい。販売中心の環境では十分に試せず、失敗を恐れて購入を避ける人も多い。

・本サービスでの解決アプローチ

香料を一つずつ試しながらブレンドできる体験を提供し、スタッフのサポートで自分らしい香りを見つけられる仕組みを作る。

■ 課題②：香水への心理的ハードルが高い

・現状の問題点（なぜ課題なのか）

香水は「上級者向け」「センスが必要」「高価」といったイメージが強く、初心者が気軽に挑戦しにくい。

・本サービスでの解決アプローチ

初心者歓迎の体験型ワークショップとして、分かりやすい工程と少人数サポートで安心して参加できる環境を整える。

■ 課題③：思い出として残る体験が少ない

・現状の問題点（なぜ課題なのか）

観光やデートでは消費型の体験が多く、形として残る思い出が少ない。特別感や共有価値が不足している。

・本サービスでの解決アプローチ

世界に一つの香りを作る体験を通じて、記憶と形の両方に残る思い出を提供し、記念日や旅行の特別な時間を演出する。

3 提案するソリューション

サービスの特長 必須
競合と差別化するサービスの特長を3つ

■ 特長①：ストーリーブレンド体験

・説明（50〜80字）

香りを選ぶだけでなく、テーマや思い出から逆算して調香する独自のストーリー設計型ワークショップ。

・ユーザーにとってのメリット

自分らしさや記念日の意味を込めた、感情に残る香水を作れる。

■ 特長②：少人数パーソナルサポート

・説明（50〜80字）

少人数制でスタッフが一人ひとりに寄り添い、香り選びから配合まで丁寧にサポートする体験設計。

・ユーザーにとってのメリット

初心者でも安心して参加でき、失敗の不安なく理想の香りに近づける。

■ 特長③：思い出を形に残すデザイン性

・説明（50〜80字）

完成ボトルやパッケージにもこだわり、写真映えするデザインと持ち帰り体験を重視。

・ユーザーにとってのメリット

体験だけでなく、見た目にも満足できる記念品として長く楽しめる。

Webアプリケーション（サーバーサイド）の範囲 必須
制作するWebアプリケーションの機能概要

【予約管理（CRUD）】

・概要：予約の新規登録・変更・キャンセル・ステータス管理を行う

・優先度（Must / Should / Could）：Must

【予約枠（スケジュール）管理（CRUD）】

・概要：日付/時間/定員/受付可否を管理し、カレンダーで運用できるようにする

・優先度（Must / Should / Could）：Must

【顧客管理（CRUD）】

・概要：氏名・連絡先・来店履歴・メモ・タグを管理する

・優先度（Must / Should / Could）：Must

【検索・フィルタ】

・概要：予約/顧客/枠を日時・ステータス・名前などで絞り込み検索する

・優先度（Must / Should / Could）：Must

【ログイン／権限管理】

・概要：管理者・スタッフなどの役割ごとに操作権限を制御する

・優先度（Must / Should / Could）：Must

【プラン・料金管理（CRUD）】

・概要：コース内容、価格、所要時間、表示/非表示を管理する

・優先度（Must / Should / Could）：Should

【通知（リマインド/キャンセル連絡）】

・概要：予約確定・前日リマインド・キャンセル等の自動/手動通知を行う

・優先度（Must / Should / Could）：Should

【レポート／集計】

・概要：予約数・キャンセル率・売上・チャネル別などを可視化する

・優先度（Must / Should / Could）：Should

【顧客メモ／対応履歴】

・概要：好み・アレルギー・要望などの記録と引き継ぎを行う

・優先度（Must / Should / Could）：Should

【CSVエクスポート／インポート】

・概要：予約・顧客データの出力と一括登録を行う

・優先度（Must / Should / Could）：Could

【在庫・資材管理】

・概要：ボトル・香料・資材の残数を記録し欠品を防ぐ

・優先度（Must / Should / Could）：Could

【監査ログ】

・概要：誰がいつ何を変更したかを記録し運用トラブルを防ぐ

・優先度（Must / Should / Could）：Could
