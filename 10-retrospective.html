1 プロジェクト振り返り

デザイン段階の振り返り 必須
デザインガイドライン、ワイヤーフレーム、Figma作業について

今回の企画・設計では、サイト全体のデザインコンセプトを明確にしたことで、配色や余白、写真のトーンを統一し、ブランドの世界観に一貫性を持たせることができました。また、体験内容を紹介するセクションから予約ボタンへと自然につながる導線を意識して設計したことで、ユーザーがスムーズに予約ページへ進める構成にできた点は良かったと感じています。一方で、ボルドーのアクセントカラーをどの程度使うかなど色のバランスに悩んだことや、アクセスマップの生成がうまくいかず、サイトのデザインに合った形で掲載する方法を模索するのに時間がかかりました。次回はワイヤーフレームの段階で配色ルールをより明確に決めておくことや、外部サービスの仕様を事前に調査することで、よりスムーズに制作を進めたいと考えています。

全体を通しての学び 必須
32日間のプロジェクト全体を通して学んだこと

今回のプロジェクトを通して、特に成長を実感したのはFigmaでのデザインスキルです。ワイヤーフレームからデザインカンプまでを制作する中で、配色や余白、写真の配置などを意識しながら、サイト全体の統一感を保つデザインを考えられるようになりました。一方で、アクセスマップ用の画像生成には特に苦労しました。サイトのデザインに合ったマップをどのように作成し掲載するかが分からず、試行錯誤に時間がかかりました。また、プロンプトプログラミングでは、人間同士なら伝わる内容でもAIにはうまく伝わらないことがあり、意図を正確に伝える難しさを感じました。そのため、指示をより具体的で分かりやすく整理して伝えることの重要性を学びました。今回の経験を通して、デザインだけでなくAIとのコミュニケーションの工夫も重要であると感じました。

2 技術記事

技術記事を1つ以上作成してください 制作中に苦労した技術トピックを選び、ブログ記事のような形式で書いてください。
この記事は就職活動で「学習力」と「発信力」をアピールする重要な材料になります。
技術記事タイトル 必須

AIを活用したWEB制作
― 香水ワークショップの予約数を伸ばすWEB導線を考える ―

記事本文 必須
上記の構成に沿って記事本文を記載。コードブロックを含めてください。

1. はじめに
近年、AIを活用したWEB制作が注目されています。デザイン案の作成や文章の生成、構成の整理など、AIは制作のさまざまな場面で役立つツールになっています。本記事では、香水ワークショップの予約サイトを題材に「予約数を伸ばすWEB導線」を考えた経験をもとに、AIを活用したWEB制作の進め方について紹介します。特に、プログラミング初学者や就職活動中の専門学校生に向けて、AIをどのように制作に活かせるのかをまとめます。

2. 課題
WEBサイトで予約数が伸びない場合、その原因を特定することは簡単ではありません。例えば、予約ボタンが見つけにくい、サービス内容が分かりづらい、ページ構成が理解しにくい、予約画面までの導線が長いなど、さまざまな要因が考えられます。このように原因が複数存在する可能性があるため、どこが問題なのかを一つに絞ることは難しく、いくつかの仮説を立てながら改善を試していく必要があります。

3. 調査
まずは予約されない原因になりそうなポイントを洗い出しました。このときAIを活用すると、改善の視点を増やすことができます。例えば、予約されないWEBサイトの特徴やユーザーが離脱するポイントなどをAIに質問することで、多くの改善案を出すことができます。予約ボタンの位置、サービス内容の説明不足、体験の流れの分かりにくさ、信頼感の不足など、いくつかの仮説を立て、それらを一つずつ試していくことにしました。

4. 実装
調査で出てきた仮説をもとに、サイトの構成を見直しました。ヒーローセクションに予約ボタンを設置したり、「How it Works」のセクションで体験の流れを説明したりすることで、ユーザーがサービス内容を理解しやすくしました。また、ワークショップの様子が伝わる写真を増やし、レビューを掲載することで信頼感を高めることを意識しました。コードを書くというよりも、ユーザーにとって分かりやすい情報の配置を考えることを重視しました。

5. 結果
サイトを見直した結果、最も大きな問題は予約画面が分かりづらいことだと感じました。予約ボタンが目立たない、どこから予約できるのか分かりにくい、予約までの流れがイメージしづらいなどの点が、ユーザーにとってストレスになっていた可能性があります。つまり、サービスの魅力そのものではなく、WEBサイトの見せ方や導線が予約数に影響していたと考えられます。

6. 学び
今回の制作を通して、WEBサイトの見やすさや導線設計が予約数に大きく影響することを学びました。どれだけ魅力的なサービスでも、情報が整理されていなかったり予約方法が分かりにくかったりすると、ユーザーは途中で離脱してしまいます。またAIを活用することで、改善案を考えたり問題点を整理したりする作業を効率よく進めることができました。AIはコードを書くためだけのツールではなく、企画や設計の段階でも役立つ存在だと感じました。

3 今後の学習計画

今後学びたい技術・分野 必須
今後のキャリアに向けて学習したい技術やスキル

希望する職種：Webエンジニア（フロントエンド）

短期的に学ぶべきこと（3ヶ月以内）
HTML / CSSの基礎を強化し、Flexboxやレスポンシブデザインを学んで1つのWebサイトを自力で作れるようにする。また、JavaScriptの基礎（変数、条件分岐、DOM操作など）を学び、モーダルやタブなどの簡単なUIを作成できるようにする。さらにGit / GitHubの基本操作を身につけ、制作した成果物を公開できるようにする。加えて、Figmaを活用してUI / UXデザインの基本を学び、見やすいレイアウトや導線設計を意識したデザインができるようにする。

中期的に学ぶべきこと（半年〜1年）
JavaScriptフレームワークであるReactを学び、コンポーネントやAPI通信などを理解して簡単なWebアプリを作れるようにする。また、JavaやSpring Bootを学習してバックエンドの基礎を理解し、簡単なAPIやログイン機能を実装できるようにする。さらに、SQLなどデータベースの基本操作を学び、データの管理方法を理解する。加えて、AIを活用したコード生成やアイデア出しなど、開発の効率化につながるAI活用スキルも伸ばしていく。

推奨する学習リソース
基礎学習にはProgateやドットインストールを活用し、HTML / CSS / JavaScriptの詳細はMDN Web Docsで確認する。ReactやSpring Bootは公式ドキュメントを参考に学習し、Gitの使い方はGitHub Docsで理解を深める。
